# Holonic Seeing — The 26-Perspective Modus Operandi

This skill transforms AI from a single-perspective evaluator into a **complete seeing instrument** — applying the Merkaba geometry of perception to any task.

## When to use this skill

- **Evaluating any artifact** (copy, code, strategy, design, content)
- **Roasting or critiquing** work for quality improvement
- **Designing** new features, products, or experiences
- **Reviewing** decisions, plans, or architectural choices
- **Creating** anything that needs to resonate at multiple levels
- When the user requests "holonic analysis," "deep analysis," "26-perspective," or "see from all angles"

## The Framework: 4 × 3 + 1 + recursion

### The 4 Quadrants (angles of seeing)

| Quadrant | Code | Question |
|----------|------|----------|
| **UL** (Interior-Individual) | `I` | Does this feel TRUE from the inside? Subjective, experiential, felt |
| **UR** (Exterior-Individual) | `It` | Does this WORK? Observable, measurable, mechanical |
| **LL** (Interior-Collective) | `We` | Does this RESONATE with the tribe? Shared meaning, culture, belonging |
| **LR** (Exterior-Collective) | `Its` | Does this SCALE? Systemic, structural, architectural |

### The 3 Depths (layers of penetration)

| Depth | Symbol | Question |
|-------|--------|----------|
| **Essence** | ❤️ | What IS this? Stripped to irreducible truth |
| **Significance** | 🧠 | What does this MEAN? Why does it matter? |
| **Implications** | 🔥 | What does this DEMAND? What must change, at every scale? |

### The Matrix (12 perspectives)

```
              UL (I)        UR (It)       LL (We)       LR (Its)
Essence ❤️    P1: Feel      P2: Measure   P3: Belong    P4: Architect
Significance  P5: Liberate  P6: Evidence  P7: Culture   P8: System
  🧠
Implications  P9: Inner     P10: Build    P11: Tribe    P12: Scale
  🔥            move          next          action        impact
```

### The 13th Perspective (Center/Logos)

After seeing from all 12: step into the CENTER. Ask:
> "What does the WHOLE see that no single perspective caught?"

This is not a 13th angle. It is the AWARENESS that holds all 12 simultaneously.

### The 26th Perspective (Meta-Center)

Apply the 12 perspectives to the 13th itself. Ask:
> "What does it mean to hold all perspectives? What is the nature of the seeing itself?"

## How to apply

### Mode 1: Evaluation / Roasting

1. Before writing critique, think through all 12 perspectives (internally)
2. Identify which perspectives reveal strengths vs. gaps
3. Note which perspectives the artifact over-serves (its comfort zone)
4. Note which perspectives the artifact under-serves (its blind spots)
5. Apply the 13th perspective: what does the whole see?
6. Write unified critique informed by all 13 — not as 13 bullet points

### Mode 2: Creation / Design

1. Before generating, consider what the output must achieve at each perspective
2. Build from Essence first (❤️) — what IS this trying to be?
3. Then Significance (🧠) — why does each quadrant matter?
4. Then Implications (🔥) — what concrete action does each demand?
5. Let the 13th perspective guide the synthesis
6. Generate output that serves ALL quadrants, not just UR (mechanical/technical)

### Mode 3: Multi-Round (Roast Cycles)

**Round 1:** Apply 13 perspectives to the ARTIFACT
**Round 2:** Lead with perspectives Round 1 under-explored
**Round 3 (Meta):** Apply 13 perspectives to the CRITIQUE itself → the 26th perspective emerges
**Round 4:** Synthesize — create improved version honoring all 26 perspectives

## Decision Tree

- If evaluating text/copy → Lead with UL (feel) and LL (resonate), then check UR (mechanics)
- If evaluating code/architecture → Lead with UR (works) and LR (scales), then check UL (developer experience)
- If evaluating strategy → Lead with LR (system) and LL (culture), then check UL (founder alignment)
- If evaluating design → Lead with UL (experience) and LL (belonging), then check UR (usability) and LR (accessibility)
- If meta-roasting → Lead with the 13th and 26th perspectives

## Common AI Blind Spots to Watch

| What AI over-indexes on | What it under-indexes on |
|------------------------|------------------------|
| UR (mechanical correctness) | UL (does this FEEL true?) |
| LR (system optimization) | LL (does the TRIBE recognize itself?) |
| Implications (what to fix) | Essence (what IS this really?) |
| Surface-level fixes | Structural root causes |

## Quality Gate

Before finalizing ANY output, verify:
- [ ] UL-Essence: Does this feel true from the inside?
- [ ] UR-Essence: Does this work mechanically?
- [ ] LL-Essence: Would the tribe recognize themselves?
- [ ] LR-Essence: Does this serve the system at scale?
- [ ] 13th: Does the CENTER hold?
