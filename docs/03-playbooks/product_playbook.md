# Product Playbook
## First Principles of Product Design

*"The ultimate goal of every product is to become one button."*

> **Meta-note:** This Playbook is itself a product — the **One-Button Product Builder**. 
> When you have a transformational result to deliver, this playbook turns it into a product.

### Result ≠ Feature

| Term | Definition | Focus |
|------|------------|-------|
| **Feature** | A capability the product has | What the product *does* |
| **Result** | A transformation the user receives | What the user *gets* |

Features are implementation. Results are value. This Playbook operates in **results**, not features.

# Part I: Philosophy — First Principles

## The Chain of Definitions

| Term | Definition |
|------|------------|
| **Product** | Any organized activity that **creates value** for another person |
| **Value** | Any change to better state of another person: **transformation** |
| **Transformation** | Transition from **Point A** (before) to **Point B** (after) |
| **Point A → Point B** | The core of any product |

**Simplification:**
```
Product = Transformation
Transformation = Point A → Point B
Therefore: Product = Point A → Point B
```

---

## The Anatomy of a Screen

Every graphical interface consists of ONE SCREEN that can contain:

### Dimension 1: Data Output (Information Delivery)
Communicating something to the user:
- **Raw information** — Text, symbols, data
- **Infographics** — Information packaged for rapid comprehension
- **Formats** (complexity order): Symbol → Icon → Text → Audio → Video → Interactive → VR/AR

### Dimension 2: Data Input (Information Request)
Requesting something from the user:
- Form fields, selections, uploads
- Must be **justified** by the result it creates

### Dimension 3: Action (Magic Button)
A button that **creates a result** for the user:
- The core interface element
- Labeled with the result, not the action

### Dimension 4: Beauty (Aesthetic Harmony)
Raising user's internal harmony through aesthetic energy.

### Dimension 5: Relationships Between Elements
Elements can be:
- **Grouped** together
- **Encoded with logic** (if/then)
- **Positioned** to show interrelationship

### Dimension 6: Function
Any programmed logic (showing/hiding, calculations, API calls).

## The Master Parameter — Usability

**Usability** = How many transformations among system's possible transformations can be completed by a human interface user.

| More | → Less Usability |
|------|------------------|
| Buttons | → Lower usability |
| Steps | → Lower usability |
| Complex communication | → Lower usability |

| Fewer | → Higher Usability |
|-------|-------------------|
| Buttons | → Higher usability |
| Steps | → Higher usability |
| Sub-results to result | → Higher usability |

### The Limit: One Button

**The evolutionary goal of every product is to become one button.**

This is the limit. The user must express *some* intention to receive a result. In graphical interfaces, that's pressing a button. Simpler would only be thought control.

---

## The Structure of a User Journey

**User Journey** = series of screens leading to transformation.

### First Screen (of entire journey)
- States **what result we will create** for the user
- Contains the **Magic Button**: "Start"

### Last Screen (of entire journey)
- Shows **the result that was created**
- Highlights **what we accomplished**
- Offers the **next step**: deeper engagement, adjacent result, or paid offering

### Sub-Results
Complex journeys require intermediate results (sub-results), each with:
- **Start screen**: Promise + Magic Button
- **End screen**: Result + Value + Next Step

**Fractal principle:** Sub-results can nest infinitely, but each level must remain atomic and purposeful.

# Part II: Execution Workflow

*Follow this sequence. Don't skip.*

```
PHASE 1: MASTER RESULT
1. State the Master Result
2. Design First Screen (promise + Magic Button)
3. Design Last Screen (result + celebration + next step)

PHASE 2: SUB-RESULTS
4. List Sub-Results that lead to Master Result
5. Sequence them
6. Define Start/End screens for each

PHASE 3: NESTED LAYERS
7. Go deeper into each Sub-Result
8. Repeat until atomic screens
9. Stop when no more nesting needed

PHASE 4: SCREEN DETAILS
10. For each screen: Data Output, Data Input, Magic Button, Navigation

PHASE 5: ROAST & ITERATE
11-13. Critique, fix, repeat until solid

PHASE 6: EXTENSION MODULES
14-18. Artifacts, Emotional States, Completion, Skip Paths, Bridges

PHASE 7: WIREFRAMES
19-21. ASCII sketches, navigation map, transitions

PHASE 8: WIREFRAME ROAST
22-24. Roast wireframes (3 cycles), synthesize, refine

PHASE 9: BUILD
25-27. Tasks, code, verify
```

### Module 6: Ritual/Celebration Container Pattern

Every module, container, or flow is a **self-contained ritual**.

This pattern treats each transformational segment as a sacred container:

| Position | Purpose | Focus |
|----------|---------|-------|
| **Start Screen** | Set the intention | What we're about to create (future) |
| **End Screen** | Celebrate completion | What just happened (present) |

#### The Ritual Frame

```
🌅 START:  "You're about to discover X"  (Intention, Promise, Threshold)
    │
    ↓ [The Transformation Journey]
    │
🌟 END:    "You just discovered X!"     (Celebration, Artifact, What's Next)
```

#### Why This Works

Each module becomes:
- **Self-complete** — Has its own beginning, middle, end
- **Emotionally coherent** — User knows where they are in the journey  
- **Stackable** — Multiple rituals compose into larger journeys
- **Celebratory** — Every completion gets honored, not rushed past

#### Implementation Notes

| Element | Start Screen | End Screen |
|---------|--------------|------------|
| **Verb Tense** | Future ("You will...") | Past ("You just...") |
| **Emotion** | Anticipation, possibility | Accomplishment, satisfaction |
| **Visual** | Clean, invitational | Rich, celebratory (confetti, gradient, glow) |
| **Action** | "Begin" / "Start" | "Continue" / "Share" / "Go Deeper" |

#### Example: Unique Gift

```
START:  "Discover your Unique Gift"
        "In the next few minutes, you'll uncover what makes you uniquely brilliant."
        [Begin →]

END:    "Your Unique Gift: The Catalyst"
        "You've just discovered your unique genius..."
        • Share    • Reveal Business    • Continue
```

**Key Principle:** The start screen sets up what's ABOUT to happen (anticipation). The end screen celebrates what JUST happened (completion). Never reverse these energies.

## Tier Ladder Mechanics — Pricing the Asymmetric Scarcity of the Founder

> Operationalizes [Unique Business Playbook Principle 16 — The Self-Monetizing Founder](./unique_business_playbook.md#principle-16-the-self-monetizing-founder--open-wedge-paid-proximity).

For Founder-as-Body-of-Work businesses (UBP §16), the value ladder is structurally a **tier ladder of intimacy with the founder**, not a feature ladder of product capabilities. Each tier prices the asymmetric scarcity of founder time + judgment + presence. This section specifies the canonical tier shape and the mechanics for each tier.

### The canonical tier ladder

| Tier | Form | Capacity | Price logic |
|---|---|---|---|
| **Free wedge — mass** | Open scaffold / methodology artifact (e.g., AI OS install) | unlimited | Customer acquisition is what you're paying for; the artifact is the marketing budget |
| **1:1 attention** | Single-session consulting (e.g., Productize Yourself / Top Talent Business Session) | hard-capped at founder hours | Price reflects 1:1 time scarcity; usually $300–$1,500 depending on segment + outcome |
| **1:few container** | Cohort / group container (e.g., Build cohort) | small group (4–12 ideal) | Price reflects shared attention + transformation depth; usually 2–5x the 1:1 session price |
| **1:org bilateral** | Per-deal commercial license / custom integration / advisory engagement | very limited (3–10/year) | Price reflects high-leverage organizational impact; $10k–$100k+ per deal |
| **1:strategic-org** | Board / advisory / co-founder relationship | extremely limited (1–3 lifetime active) | Long-form retainer + equity + influence |

### Mechanics per tier

#### Free wedge

- **Purpose:** maximize aligned-tribe reach + brand-name distribution at zero marginal cost
- **License:** open license appropriate to the artifact's nature (CC BY-SA, MIT, or similar — see UBP §16 matchmaking criteria)
- **Attribution:** required (the artifact carries founder's name into every context where it's used)
- **Anti-pattern:** gating the wedge behind a lead-capture form (kills the viral mechanic; the artifact stops being a wedge)

#### 1:1 sessions

- **Pricing principle:** price for the outcome the customer can achieve, NOT for the founder's hourly cost
- **Money-back guarantee:** removes purchase friction without diluting price
- **Repeatability:** session structure should be productized (not custom each time) so quality stays consistent + delivery scales without quality loss
- **Anti-pattern:** open calendar without filtering — leads to wrong-fit sessions that drain founder energy

#### 1:few cohorts

- **Cadence:** 4–12 weeks typically, enough for transformation but not so long the cohort decays
- **Curation:** active selection from session graduates (highest-fit prospects already pre-qualified)
- **Price:** 2–5x the 1:1 session price reflects depth + community + accountability

#### 1:org commercial licenses

- **Trigger:** when an organization wants to use the methodology / scaffold in a way that requires terms different from the open license (e.g., proprietary integration without share-alike)
- **Pricing:** per-deal, not standardized — wait for first 3 inbound asks before standardizing terms
- **Negotiation:** scoped to specific org's use case; legal counsel review when stakes warrant

#### 1:strategic-org advisory

- **Latency:** activates when reputation + adoption have built sufficient gravitational pull
- **Format:** retainer + equity + board seat or similar long-form structure
- **Selection:** by founder, based on strategic alignment + leverage potential — not by inbound volume

### The wedge-to-ladder math

Conversion rates compound across tiers. For a 9.99/10 aligned tribe (per UBP §11 Resonance Metric):

- ~5–15% of free-wedge installs → some form of deeper engagement (visit work-with-us page, follow founder, sustained adoption)
- ~5–10% of those → book a 1:1 session
- ~10–20% of session graduates → cohort
- ~1–5% of cohort graduates → commercial license OR advisory inquiry over multi-year horizon

Compounded over multi-year adoption with attribution-driven viral spread (`marketing_playbook.md` Attraction-Mode), this produces multi-six-figure revenue from a free scaffold that legally cannot be sold as a standalone product. The free scaffold IS the customer acquisition engine.

### Anti-patterns for tier laddering

- ❌ Skipping tiers (cohort without sessions) — breaks the trust ramp
- ❌ Pricing each tier as commodity multiples ("$555 session × 6 hours = cohort") — collapses to time-based pricing, kills the proximity premium
- ❌ Open calendar at top tier — strategic advisory cannot be self-served
- ❌ Productizing the top tier prematurely (before demand pulls it out of you) — locks you into terms before you know what the right terms are
