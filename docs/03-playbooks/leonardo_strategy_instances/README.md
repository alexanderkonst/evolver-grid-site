# Leonardo Strategy Instances

*Per-target applications of the Leonardo strategy. Each file in this folder is a single named patron candidate, mapped against the substrate's strengths, with reply-ready content and conversation context.*

---

## The meta-pattern

This folder operationalizes **Phase Shift Domain 87 — Multi-Patron Architecture** ([`docs/01-vision/phase_shift_technology_library.md`](../../01-vision/phase_shift_technology_library.md#domain-87-multi-patron-architecture-the-leonardo-strategy-generalized-white-label-skins-as-the-patron-relationship-vehicle)).

Leonardo da Vinci had multiple patrons across his life: Medici, Sforza, Borgia, Francis I. He moved between them. Each provided different conditions. No single relationship was load-bearing. The portfolio carried him.

The modern translation: each ecosystem leader running their own skin on the Find Your Top Talent substrate is a patron-shaped relationship. The portfolio compounds. The org optimizes for adoption count, not single-deal closure.

Each file in this folder is one named patron candidate, treated as a Leonardo-style target: their context, their archetype, what they uniquely offer, what FYTT uniquely offers them, reply-ready content for outreach, conversation runway. The folder grows one file per named candidate.

## How to add a new instance

When a new patron candidate becomes credible:

1. Create `<surname_or_org>.md` in this folder.
2. Cover: who they are (1 paragraph), why they're a candidate, what they uniquely offer, what FYTT uniquely offers them, reply-ready content (DM templates, intro paragraphs, conversation openers), known conversation runway (warm intro path, mutual connections, prior touchpoints).
3. If raw factual intel on their org / platform also gets collected, the intel goes in [`docs/02-strategy/partners/<name>.md`](../../02-strategy/partners/) and the Leonardo instance file cross-links to it.
4. When the patron lands (skin shipped + commercial relationship live), move them to the production tracker in [`docs/02-strategy/white_label_strategy.md`](../../02-strategy/white_label_strategy.md).
