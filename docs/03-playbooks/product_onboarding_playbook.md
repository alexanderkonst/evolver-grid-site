# Product Onboarding Playbook

*Created Day 130 (July 20, 2026), prompted by the Practitioner Node: the platform now begets products by bundling shipped modules, and there was no standard path from idea to product. This is that path.*

The seven steps (each step gates the next):

1. **Name the transformational result** in one sentence a stranger repeats correctly. No result sentence, no product.
2. **Bundle check**: list which ALREADY-SHIPPED modules compose it. New-build required for v1 = not this playbook; that is a feature roadmap item instead.
3. **Underwrite the category** (Domain 105): what vacant category does this occupy; who else could claim it; why us.
4. **Terms-first draft** (Domain 110): ownership, rev-share, exit rights on one page BEFORE any offer conversation.
5. **First-instance proposal**: sell it to ONE named person before any landing page exists (BIG IDEA #1: the first holon tests everything). The proposal text is written for that person by name.
6. **Landing from the accepted proposal**: the landing page is the accepted proposal, generalized. Not the reverse.
7. **Shelf it**: /products entry + value-ladder placement + docs/04-products/ spec doc + roadmap line. A product that is not on the shelf does not exist (Domain 103).
