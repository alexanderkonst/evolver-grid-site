# One-Button Product Builder

> A meta-system that takes a transformational result and outputs a working software product.

---

## What Is This?

**Input:** A transformational result (Point A → Point B)

**Output:** A fully specified and built software product that delivers that transformation.

**Process:** 8 automated phases, executed by AI with human-in-the-loop at key moments.

---

## The 8 Phases

```
┌─────────────────────────────────────────────────────┐
│  1. MASTER RESULT                                   │
│     Input: "From A to B" transformation             │
│     Output: One-sentence result + one magic button  │
├─────────────────────────────────────────────────────┤
│  2. SUB-RESULTS                                     │
│     Break master into 3-5 spaces                    │
│     Each space = a sub-result                       │
├─────────────────────────────────────────────────────┤
│  3. NESTED LAYERS                                   │
│     Break sections into screens                     │
│     Each screen delivers micro-result               │
├─────────────────────────────────────────────────────┤
│  4. SCREEN DETAILS                                  │
│     Data in, data out, magic button, nav            │
│     For every single screen                         │
├─────────────────────────────────────────────────────┤
│  5. ROAST & ITERATE                                 │
│     AI critiques the spec                           │
│     Human approves/adjusts                          │
├─────────────────────────────────────────────────────┤
│  6. EXTENSION MODULES                               │
│     Artifacts, Emotions, Bridges, Skip Paths        │
│     The "feel" layer                                │
├─────────────────────────────────────────────────────┤
│  7. WIREFRAMES                                      │
│     ASCII mockups of key screens                    │
│     Navigation flow map                             │
├─────────────────────────────────────────────────────┤
│  8. BUILD & VERIFY                                  │
│     AI generates code                               │
│     Human tests and deploys                         │
└─────────────────────────────────────────────────────┘
```

---

## Why This Works

1. **First-principles foundation.** Every screen exists because it delivers a result. No bloat.

2. **Structured for AI.** Each phase has explicit inputs and outputs. AI can execute.

3. **Human-in-the-loop.** Defining the result, roasting, and taste remain human.

4. **Iterative by design.** Roast cycles catch errors before code is written.

5. **Modular.** Extension modules can be added/removed per product.
