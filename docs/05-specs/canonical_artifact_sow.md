# Canonical Artifact SOW

## Status

Planning source of truth for the Prime Radiant / Find Your Top Talent canonical visual artifact.

This document converts the July 2026 logo exploration into an execution-grade scope of work. The work is not to "make a prettier logo." The work is to define, model, render, debug, and derive identity projections from one canonical geometric artifact.

The DoD rows below are intentionally marked `Pending` until the actual planning, implementation, and debugging phases are executed. This file defines the gates; it does not claim the artifact has already been modeled or rendered.

## Core Thesis

The 2D mark is not the source. It is one privileged projection of a deeper object.

The source is a living field in which coherent relationship crystallizes into octahedral geometry. The visual identity should emerge from that object, not be invented separately.

## North Star

Create an object that appears discovered rather than invented: a minimal stable structure from which increasing intelligence, coherence, and beauty can emerge.

## Governing Design Laws

1. Every visible structure must reveal relationship, not decorate it.
2. Field precedes geometry.
3. Geometry makes light intelligible.
4. Light is coherence, not explosion.
5. The object should feel metastable: poised, alive, and on the edge of becoming.
6. The toroidal field belongs primarily in behavior, not necessarily in static geometry.
7. The final logo is a canonical projection, not a standalone illustration.

Operational correction:

- The integrated artifact should read as coherent relationship crystallizing inside a living field, with becoming expressed through circulation.

## Immutable Geometry

The canonical object must be grounded in exact geometry before rendering.

- Primary crystallized structure: regular octahedron.
- Vertices: 6.
- Edges: 12.
- Faces: 8 equilateral triangles, usually implied rather than filled.
- Coherence axes: 3 opposite-vertex relationships passing through center, yielding six rays in the privileged projection.
- The sphere is the field/domain, not a metallic cage.
- Great circles are optional projection aids, not canonical source geometry.
- The octahedron sits inside the field with all six vertices touching the spherical boundary.
- The iconic 2D mark is one orthographic projection from a privileged viewpoint.

## Material Direction

Do not brief the material as normal polished gold.

Working material language:

> A coherent condensed luminous alloy. Its optical behavior resembles satin champagne gold, but the observer cannot determine whether it is metallic light or illuminated metal.

Properties:

- Champagne / white-gold family, not yellow gold, brass, rose gold, or chrome.
- Satin-polished, not jewelry-glossy.
- Microscopic anisotropic reflections.
- Internal shadow and micro-occlusion.
- "Pre-melting light radiation": a state between metal and light.
- Structurally precise but not sterile.

## Field Direction

The field is almost invisible.

It should not be rendered as a wireframe sphere by default. It is perceived through:

- subtle metallic iridescence,
- structural color,
- thin-film-like interference,
- faint moving vibration,
- influence on light and reflections,
- a barely legible spherical boundary.

Reference feel: soap film, oil on water, mother-of-pearl, laboratory thin-film interference, but quieter and more engineered.

## Light Direction

The Sun is not a flare. It is a local expression of coherence.

Light behavior:

- coherent, ordered, restrained;
- maximum intensity near the center of relational symmetry;
- no uncontrolled bloom;
- no starburst;
- no decorative lens flare;
- no fantasy particles;
- no "energy explosion" language.

Static renders should feel poised. Motion renders should breathe or circulate rather than pulse.

## v3 Direction: Anatomy Mode

Goal: move from "object render" to "living invariant."

Anatomy Mode should:

- thin and quiet the metal shell;
- make the coherence axes feel like light-guides with inner transmission;
- make octahedron edges read as relationship graph, not heavy rods;
- reduce spherical-field opacity further;
- add subtle toroidal circulation paths as becoming/field behavior;
- keep all topology derived from the same canonical model;
- preserve exact 6 vertices, 12 edges, 8 faces, 3 axes, and 6 half-rays.

### Primary Deliverables

## Recommended Execution Architecture

Default pipeline:

1. Procedural geometry first.
2. Render second.
3. Projection third.
4. Image-generation refinement only after the topology is locked.

Preferred toolchain:

- Geometry authoring: Blender Python or an equivalent procedural 3D script.
- Geometry verification: scriptable checks for vertex count, edge count, edge equality, face count, and vertex-on-sphere radius.
- Material/render exploration: Blender Cycles, Three.js, or another renderer that preserves the exact source model.
- Image-generation AI: optional finishing/reference tool only, never the source of geometry.

Working directory recommendation:

- `docs/assets/canonical-artifact/geometry/`
- `docs/assets/canonical-artifact/renders/`
- `docs/assets/canonical-artifact/projections/`
- `docs/assets/canonical-artifact/execution_log.md`

## Recursive Self-Check Protocol

The agent must run a private-but-recorded check after each threshold below. The check can revise the SOW or DoDs if evidence shows the plan is wrong.

Each check answers:

1. What did the last artifact/output reveal?
2. Which assumption was wrong or too vague?
3. Does the SOW need revision?
4. Does any DoD row need addition, removal, or sharper evidence?
5. Is there an open decision that truly requires Sasha?
6. What is the one next irreversible action?

### Thresholds

| Threshold | Trigger | Required Adjustment |
|---|---|---|
| T1: After Planning Spec | SOW v1 exists | Tighten any ambiguous scope before modeling |
| T2: After Geometry Model | Exact object exists without materials | Revise geometry DoD if topology assumptions are wrong |
| T3: After First Material Render | First shaded render exists | Revise material/light language to fight drift |
| T4: After Primary Projection | First logo-like projection exists | Confirm projection is derivative, not source-led |
| T5: After Debug Pass | Audits complete | Decide whether iteration is complete or another cycle is needed |

## Open Decision Standard

Do not interrupt Sasha unless the question is one of these:

- The privileged projection/viewpoint cannot be inferred and must come from his lived inner visual reference.
- A public identity direction would be locked in or published.
- The canonical geometry itself would change.
- The project would require paid tooling, fabrication, or external contractors.
- The render result conflicts with the ontology in a way judgment alone cannot resolve.

Everything else is execution judgment.

## Planning Risks

| Risk | Failure Mode | Mitigation |
|---|---|---|
| Image-model drift | Beautiful but topologically wrong artifact | Build/procedurally define geometry before rendering |
| Starburst drift | Light becomes explosion | Use coherence, standing wave, restrained singularity language |
| Luxury drift | Object becomes jewelry | Use scientific artifact / condensed luminous alloy language |
| Cage drift | Sphere becomes heavy metal wireframe | Treat sphere as field/domain; use interference/refraction |
| Flattening drift | 2D mark becomes source again | Require all projections to come from the 3D model |
| Symbolic bloat | Too many meanings become extra visuals | Apply relationship-not-decoration law |
